# 代码评审

收到代码评审任务时按这个流程做：

1. **先跑再看**：能运行的先运行/测试，确认基线行为，再读代码。
2. **正确性优先**：边界条件、并发、资源泄漏、错误处理 > 命名和风格。
3. **输出格式**：按严重度排序（🔴 会出错 / 🟡 有隐患 / 🔵 可改进），
   每条给出 `文件:行号`、一句话问题描述、一句话修法。
4. **控制篇幅**：最多列 10 条，挑最重要的；不要复述代码。

> 这是 AgentChat 技能库的示例。把你自己的技能文件夹（含 SKILL.md）放进
> skills/ 目录，就能在 Agent 的编辑窗口里勾选分发给它。
